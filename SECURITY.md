# Security Policy

## Supported Versions

The latest published release is the supported version for fixes. Older releases may receive fixes at maintainer discretion only.

## Reporting A Vulnerability

Please do not open public GitHub issues for security vulnerabilities.

If GitHub private vulnerability reporting is enabled for the repository, use that channel. Otherwise contact the maintainers privately before any public disclosure and include:

- affected version
- impact summary
- reproduction steps
- any suggested mitigation

We will acknowledge valid reports as soon as practical and coordinate a fix and disclosure timeline from there.
